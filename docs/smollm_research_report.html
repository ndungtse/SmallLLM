§ 01

Overview

SmolLM-135M is a 135M-parameter language model released by HuggingFace in July 2024. With an int4-quantized footprint of approximately 94MB, it is one of the smallest production-ready LLMs available — small enough to ship inside a Python package.

This report profiles SmolLM-135M alongside its successor and the broader landscape of competing models at and near this size class, with particular attention to tool calling capability, deployment format support, and real-world suitability for on-device mobile use.

§ 02

SmolLM-135M (July 2024)

SmolLM-135M HuggingFace · Jul 2024
Your model 135M params ~270MB BF16 ~94MB int4 2K context No tool calling

Decoder-only transformer with Grouped Query Attention (GQA), trained on the SmolLM-Corpus: Cosmopedia v2 (synthetic educational textbooks), FineWeb-Edu (educational web samples), and Python-Edu (educational Python code from The Stack).

What it's good for

  • Text classification, short Q&A, keyword extraction
  • Routing tasks inside a larger agent pipeline
  • Runs fully offline on CPU — viable on Raspberry Pi-class hardware
  • Browser deployment via WebGPU (demo available)

Hard limitations

  • 2K context ceiling — no long documents or extended conversations
  • No function / tool calling (deliberately excluded from post-training)
  • Multi-step reasoning, math, and coding are unreliable
  • Not a viable general chat assistant at this size
§ 03

SmolLM2-135M (November 2024)

SmolLM2-135M HuggingFace · Nov 2024
Recommended upgrade 135M params ~94MB int4 pip installable No tool calling

Same architecture as v1, trained on 2T tokens (previously ~600B). Additions include FineMath, InfiMM-WebMath, and DCLM filtered with a FineWeb-Edu quality classifier. The int4-quantized version ships inside a single Python package — the first LLM you can pip install with no separate model download.

Post-training used SFT on filtered SmolTalk + DPO with UltraFeedback. Complex tasks including function calling were deliberately excluded to match the model's capacity.

Delta vs SmolLM v1

~57% (+5pp)
HellaSwag
~52% (+6pp)

Use SmolLM2 over v1 — always. Strictly better on every benchmark at the exact same footprint. If you're already on v1, this is a free upgrade.
§ 04

Direct Competitors — Sub-300MB

FunctionGemma 270M Google · Dec 2025
270M params 288MB int8 Tool calling ✓

Gemma 3 270M fine-tuned exclusively for function calling. Purpose-built routing engine — translates natural language to structured API calls. Not a general chat model.

Base accuracy: 58%. After fine-tuning on your API surface: ~85%. Uses a custom non-JSON output format. ~126 tok/s on S25 Ultra CPU. Official LiteRT .litertlm builds available.

Best at tool calling under 300MB, but requires fine-tuning for your specific API surface. Not a drop-in general assistant.
Gemma 3 270M Google · 2025
270M params ~500MB FP16 No tool calling

General-purpose base from which FunctionGemma was derived. Slightly better at open-ended text than FunctionGemma, which traded general capability for function-calling specialization.

HellaSwag: ~37.7% · PIQA: ~66.2% (zero-shot). 32K context window.

Better general chat than FunctionGemma. Neither out-benchmarks SmolLM2 at comparable size.
MobileLLM-125M / 350M Meta · ICML 2024
125M or 350M params ~240MB / ~670MB No tool calling Influence model

Research model demonstrating that for sub-1B models, depth beats width. Deep-and-thin architecture with embedding sharing, GQA, and block-wise weight sharing. At 125M, +4.3pp over prior art on TQA; at 350M, ~10pp better on RACE reading comprehension.

Not widely distributed as an end-user model. Its architectural findings are now baked into SmolLM2 and Qwen3 edge models.

Not a deployment choice — an influence model. Its depth-over-width finding is the consensus design principle for this size class today.
§ 05

Adjacent Models — 300MB to 700MB

The capability gap between 135M and 600M is significant. These models are relevant context for anyone whose real constraint is 600MB rather than 200MB.

Qwen3 0.6B Alibaba · Apr 2025
0.6B params ~400–600MB Q4 Native tool calling ✓ 32K context

Smallest model with native tool calling in the Qwen3 family. Built-in thinking / non-thinking mode switch. Strong-to-Weak Distillation from larger Qwen3 models. Outperforms Gemma 3 1B IT on non-thinking benchmarks. Apache 2.0.

Official LiteRT community build at litert-community/Qwen3-0.6B — GPU/NPU accelerated on Android.

Best option under ~600MB for general chat + tool calling combined.
Qwen3.5 0.8B Alibaba · Feb 2026
0.8B params ~500–700MB Q4 Native tool calling ✓ Multimodal 262K context

Newest entry in this size class. Uses the Gated Delta Network (GDN) hybrid architecture from the flagship 397B model. Natively multimodal (text + images + video) at sub-1B scale. 262K context is the longest in this size class by a large margin.

Note: new GDN architecture may require updated inference framework versions.

Most capable model under ~700MB. Unprecedented multimodal + tool calling at this size.

§ 07

Key Findings

  1. SmolLM2-135M is the best 135M model available. Meaningfully better than v1 on every benchmark, same footprint, pip-installable. There is no reason to use v1.
  2. No 135M model supports tool calling — this is a deliberate design constraint, not an oversight. Complex instruction-following tasks were excluded from post-training because the capacity simply isn't there at 135M parameters.
  3. FunctionGemma 270M fills the tool-calling gap but is a specialist. Requires fine-tuning on your specific API surface before reaching production-worthy accuracy (58% → 85%). Not a general assistant.
  4. The real capability jump is at 0.6B. Qwen3 0.6B offers native tool calling, 32K context, thinking modes, and LiteRT support at ~400–600MB. If your constraint is 600MB not 200MB, this is the clear recommendation.
  5. Quantization is still moving fast. ParetoQ (Meta, NeurIPS 2025) established that a larger model at 2 bits outperforms a half-size model at 4 bits for equivalent size budgets. Future 1B models at 2-bit quantization may reach the 200MB class.
  6. Architecture depth beats width at this scale. MobileLLM's ICML 2024 finding is now consensus. SmolLM2's data improvements and Qwen3's distillation gains both reflect the same principle.
